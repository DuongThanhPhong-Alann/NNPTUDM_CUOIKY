POSTMAN TEST PLAN

Base URL:
http://localhost:3000

Headers thuong dung:
- Content-Type: application/json
- Authorization: Bearer <token>

Dieu kien truoc khi test:
- Da chay server
- File .env co JWT_SECRET
- Da co du lieu mau hoac tao du lieu theo thu tu ben duoi

Tong ket so luong truong hop test Postman:
- Tong so endpoint/request co the test: 66
- Cach tinh:
  + 1 health check
  + 4 auth
  + 1 upload
  + 12 nhom CRUD x 5 request = 60
- 12 nhom CRUD gom:
  + chungcus
  + canhos
  + chungcu-images
  + canho-images
  + nguoidungs
  + cudans
  + chuho
  + tintucs
  + dichvus
  + hoadon-dichvus

- Neu tinh ca cac case thanh cong + that bai + phan quyen + loi du lieu thi tong so scenario test thuc te se hon 90 case.

==================================================
1. CAC NHOM CHUC NANG CO THE TEST TREN POSTMAN
==================================================

A. HEALTH CHECK
1. GET /health
- Muc dich: kiem tra server dang chay
- Ket qua mong doi: 200, { ok: true }


B. AUTHENTICATION
1. POST /api/v1/auth/register
- Muc dich: dang ky tai khoan cu dan
- Body:
{
  "hoTen": "Nguyen Van A",
  "email": "a@test.com",
  "matKhau": "123456",
  "soDienThoai": "0909000001"
}
- Test:
  + Dang ky thanh cong
  + Dang ky trung email -> 400
  + Thieu field bat buoc -> loi

2. POST /api/v1/auth/login
- Muc dich: dang nhap lay token
- Body:
{
  "email": "a@test.com",
  "matKhau": "123456"
}
- Test:
  + Dang nhap dung -> tra token
  + Sai email -> 401
  + Sai mat khau -> 401

3. GET /api/v1/auth/me
- Muc dich: lay thong tin user dang dang nhap
- Header: Authorization Bearer token
- Test:
  + Token hop le -> tra user
  + Khong co token -> 401
  + Token sai -> 401

4. GET /api/v1/auth/users
- Muc dich: manager xem danh sach user
- Header: Authorization Bearer token manager
- Test:
  + Manager truy cap duoc
  + Cu dan truy cap -> 403


C. AUTHORIZATION
1. Test role "Ban quan ly"
- Dung token manager goi cac route POST/PUT/DELETE cua:
  + /api/v1/chungcus
  + /api/v1/canhos
  + /api/v1/chungcu-images
  + /api/v1/canho-images
  + /api/v1/cudans
  + /api/v1/chuho
  + /api/v1/tintucs
  + /api/v1/dichvus
  + /api/v1/hoadon-dichvus
  + /api/v1/nguoidungs
- Ket qua mong doi: duoc phep

2. Test role "Cu dan"
- Dung token cu dan goi lai cac route POST/PUT/DELETE tren
- Ket qua mong doi: 403 voi cac route quan tri

3. Test quyen xem va sua user
- GET /api/v1/nguoidungs/:id
  + User xem chinh minh -> duoc
  + User xem nguoi khac -> 403
- PUT /api/v1/nguoidungs/:id
  + User sua chinh minh -> duoc
  + User sua nguoi khac -> 403

E. UPLOAD
1. POST /api/v1/upload
- Header: Authorization Bearer token manager
- Body: form-data
  + key: file
  + type: File
- Test:
  + Upload thanh cong -> 201
  + Khong co file -> 400
  + Cu dan upload -> 403
  + Khong token -> 401


F. TRANSACTION
1. POST /api/v1/chat-messages
- Route nay co transaction
- Chuc nang:
  + Tao chat message
  + Cap nhat lastMessageAt trong ChatSession
  + Neu loi se rollback
- Body:
{
  "idSession": "<chatSessionId>",
  "role": "user",
  "content": "Xin chao",
  "model": "manual"
}
- Test:
  + Tao message thanh cong
  + Session khong ton tai -> 404
  + User gui vao session khong phai cua minh -> 403


G. SOCKET CHAT
1. Tao chat session
- POST /api/v1/chat-sessions
- Body:
{
  "tieuDe": "Ho tro can ho",
  "tags": ["support"]
}

2. Lay chat sessions
- GET /api/v1/chat-sessions
- Test:
  + Manager thay tat ca
  + Cu dan chi thay session cua minh

3. Lay chat messages theo session
- GET /api/v1/chat-messages?idSession=<chatSessionId>
- Test:
  + Manager xem duoc
  + Cu dan chi xem session cua minh
  + Cu dan khong truyen idSession -> 400

4. Test Socket.IO
- Ket noi socket den server
- Gui token qua auth handshake
- Emit event: chat:join voi sessionId
- Sau do goi HTTP POST /api/v1/chat-messages
- Ket qua mong doi: nhan event chat:message


H. CAC CASE LOI QUAN TRONG NEN TEST
1. Khong gui token -> 401
2. Token sai -> 401
3. Role khong du quyen -> 403
4. Id khong ton tai -> 404
5. Body thieu field required -> 400
6. Email trung -> 400
7. Tao can ho trung maCan trong cung chung cu -> loi
8. Tao hoa don dich vu trung idCanHo + idDichVu + kyThanhToan -> loi
9. Upload khong co file -> 400
10. Login sai mat khau -> 401
